# 乐学日程获取方案

## 概述

乐学（Lexue）是北京理工大学的在线学习平台，日程获取主要通过其内置的日历功能导出 iCal 格式的日程数据。

## 核心 API

### 基础信息

- **乐学地址**: `https://lexue.bit.edu.cn`
- **需要认证**: 是（通过校园SSO登录）
- **日历导出接口**: `/calendar/export.php`

### API 调用流程

```
1. GET / → 获取 sesskey（会话密钥）
2. POST /calendar/export.php → 生成日历导出URL
3. GET {导出的URL} → 获取 iCal 格式的日程数据
```

### 详细步骤

#### 第一步：获取 sesskey

```kotlin
@GET("/")
suspend fun get(): Response<GetLexueIndexDataModel.Response>
```

响应中包含 HTML，从 HTML 中用正则提取 sesskey：
```kotlin
val sesskey = Regex("[\"|']sesskey[\"|']:[\"|']([^\"|']+?)[\"|']").find(html)?.groupValues?.get(1)
```

#### 第二步：获取日历导出 URL

```kotlin
@FormUrlEncoded
@POST("/calendar/export.php")
suspend fun getCalendarUrl(
    @Field("sesskey") sesskey: String,
    @Field("_qf__core_calendar_export_form") qfCoreCalendarExportForm: String = "1",
    @Field("events[exportevents]") events: String = "all",
    @Field("period[timeperiod]") period: String = "recentupcoming",
    @Field("generateurl") generateurl: String = "获取日历网址",
): Response<GetCalendarUrlDataModel.Response>
```

#### 第三步：获取 iCal 日历数据

```kotlin
@GET
suspend fun getCalendar(
    @Url url: String,
): Response<GetCalendarDataModel.Response>
```

## 数据模型

### 日历事件 (CalendarEvent)

```kotlin
data class CalendarEvent(
    val uid: String,           // 事件唯一标识
    val event: String,         // 事件标题 (SUMMARY)
    val description: String,   // 事件描述
    val course: String,         // 课程分类 (CATEGORIES)
    val time: LocalDateTime    // 开始时间
)
```

使用 `ical4j` 库解析 iCal 格式：

val cal = CalendarBuilder().build(html.byteInputStream())
cal.components.forEach { calendarComponent ->
    val start = calendarComponent.getProperty<DtStart>(Property.DTSTART)

## 完整调用示例

```kotlin
suspend fun getLexueSchedule(): List<CalendarEvent> {
    // 1. 获取 sesskey
    val getResponse = api.schoolLexue.get()
    val sesskey = getResponse.body()?.sesskey ?: throw Exception("获取 sesskey 失败")

    // 2. 获取日历导出 URL
    val response = api.schoolLexue.getCalendarUrl(sesskey)
    val calendarUrl = response.body()?.url ?: throw Exception("获取日历URL失败")

    // 3. 获取并解析日历
    val calendarResponse = api.schoolLexue.getCalendar(calendarUrl)
    return calendarResponse.body()?.calenders ?: emptyList()
}
```

## 认证要求

- 需要使用校园网账号登录 SSO (`https://sso.bit.edu.cn`)
- 登录后会获取 cookies，需要在请求中保持 session
- 建议使用与乐学相同的 OkHttpClient 实例以共享 cookies

## SSO 认证流程

### 基础信息

- **SSO 地址**: `https://sso.bit.edu.cn`
- **登录接口**: `/cas/login`
- **账号类型**: 学号
- **密码加密**: AES/ECB/PKCS5Padding

### 认证流程

```
1. GET /cas/login → 初始化登录，获取 salt 和 execution
2. POST /cas/login → 提交登录表单，返回成功/失败
3. 使用登录后的 Cookie 访问乐学等其他服务
```

### 第一步：初始化登录

```kotlin
@GET("/cas/login")
suspend fun initLogin(): Response<GetSchoolInitLoginDataModel.Response>
```

解析响应获取关键参数：
- `salt`: AES 加密密钥（Base64 编码）
- `execution`: 表单执行 ID
- `ifLogin`: 是否已登录

```kotlin
val doc = Jsoup.parse(html)
val salt = doc.select("#login-croypto").first()!!.childNode(0).toString()
val execution = doc.select("#login-page-flowkey").first()!!.childNode(0).toString()
val ifLogin = html.indexOf("用户名密码") == -1  // 已登录则不显示登录表单
```

### 第二步：密码加密

使用从初始化获取的 `salt` 对密码进行 AES 加密：

```kotlin
fun encryptPassword(password: String, loginCrypto: String): String {
    val decodedKey = Base64.getDecoder().decode(loginCrypto)
    val secretKey = SecretKeySpec(decodedKey, "AES")
    val cipher = Cipher.getInstance("AES/ECB/PKCS5Padding")
    cipher.init(Cipher.ENCRYPT_MODE, secretKey)
    val encryptedBytes = cipher.doFinal(password.toByteArray(Charsets.UTF_8))
    return Base64.getEncoder().encodeToString(encryptedBytes)
}
```

### 第三步：提交登录

```kotlin
@FormUrlEncoded
@POST("/cas/login")
suspend fun login(
    @Field("username") username: String,
    @Field("password") password: String,          // 加密后的密码
    @Field("execution") execution: String,          // 从初始化获取
    @Field("croypto") salt: String,                // 从初始化获取
    @Field("captcha_payload") captchaPayload: String = "{}",  // 可为空
    @Field("type") type: String = "UsernamePassword",
    @Field("geolocation") geolocation: String = "",
    @Field("captcha_code") captcha: String = "",   // 不请求验证码则为空
    @Field("_eventId") eventId: String = "submit",
): Response<PostSchoolLoginDataModel.Response>
```

### 验证码说明

**关键发现**：不请求验证码接口即可跳过验证码验证

- 正常流程会先请求 `/cas/api/protected/user/findCaptchaCount/{学号}` 检查是否需要验证码
- 发现需要验证码后请求 `/cas/api/captcha/generate/DEFAULT` 获取验证码图片
- **但只要不请求验证码相关接口，直接发送登录请求就不会触发验证码验证**

```kotlin
// 不需要验证码的关键：
// 1. 不请求 /cas/checkNeedCaptcha
// 2. 不请求 /cas/getCaptcha
// 3. 直接发送登录请求，captcha_code 留空
```

### 完整登录示例

```kotlin
suspend fun loginSchool(username: String, password: String): Boolean {
    // 1. 初始化登录
    val initResponse = api.schoolUser.initLogin().body() ?: throw Exception("初始化失败")
    if (initResponse.ifLogin) return true  // 已登录

    val salt = initResponse.salt ?: throw Exception("获取 salt 失败")
    val execution = initResponse.execution ?: throw Exception("获取 execution 失败")

    // 2. 加密密码
    val cryptPassword = encryptPassword(password, salt)

    // 3. 提交登录
    val response = api.schoolUser.login(
        username = username,
        password = cryptPassword,
        execution = execution,
        salt = salt,
        captchaPayload = encryptPassword("{}", salt),
        captcha = "",  // 不需要验证码
    ).body() ?: throw Exception("登录响应错误")

    return response.success
}
```

### Cookie 处理

登录成功后，SSO 会设置以下关键 Cookie：
- `MOD_AUTH_CAS`: 主认证 Cookie
- 其他会话 Cookie

**重要**：登录后需要将 Cookie 保存并在后续请求中复用，建议使用同一个 `OkHttpClient` 实例。

### 检查登录状态

```kotlin
suspend fun checkLogin(): Boolean {
    val response = api.schoolUser.initLogin().body() ?: return false
    return response.ifLogin  // true 表示已登录
}
```

## 依赖库

- **Retrofit**: 网络请求
- **ical4j**: iCal 格式解析
- **Jsoup**: HTML 解析

## 注意事项

1. 日历导出 URL 有时效性，每次获取需要重新生成
2. `period` 参数可选择导出范围，如 `recentupcoming`（近期事件）
3. 需要处理登录态失效的情况
